# UI レビューチェックリスト

**目的**: デプロイ前のUI品質保証チェック
**ペルソナ**: テック系長期投資家 タケシ（28-42歳・IT勤務・スマホ/PC両用）

---

## 1. テキスト可読性（必須 / 論外チェック）

- [ ] **WCAG AA コントラスト比**: 全テキストが背景に対して4.5:1以上（大文字は3:1以上）
- [ ] **グラデーションテキスト**: 最も薄い色が背景に対して4.5:1以上を確保
- [ ] **muted-foreground**: セマンティック変数 `--muted-foreground` が14px日本語で十分読める（推奨7:1以上）
- [ ] **カード内テキスト**: card背景(`--card`)に対してcard-foreground/muted-foregroundのコントラスト確認
- [ ] **フォントサイズ**: 本文14px以上、モバイルで最小12px

## 2. カラーハーモニー

- [ ] **ブランドカラー統一**: `primary-500`〜`primary-700`をメイングラデーションに使用
- [ ] **セマンティックカラー使用**: ハードコード色（`text-gray-*`等）ではなくCSS変数/セマンティッククラスを使用
- [ ] **ダークモード整合性**: ライト/ダーク両方で色が破綻しないか確認
- [ ] **アクセントカラーの節度**: success/danger/warningは必要な箇所のみに限定

## 3. 視覚的リズム・レイアウト

- [ ] **セクション区切り**: 交互に背景色を変えてセクションを区別（白 ↔ muted/40〜50）
- [ ] **余白の一貫性**: section間のpaddingが統一（py-12 sm:py-16）
- [ ] **カード区別**: 白背景上のカードに十分なborder/shadowで区別可能
- [ ] **情報密度**: 同一コンポーネントの過剰な繰り返しがないか（例: 同じCTAが3回以上）

## 4. ペルソナ整合性

- [ ] **フィンテック信頼感**: 青基調 + クリーンなレイアウト + セキュリティ訴求
- [ ] **日本語最適化**: 日本語フォント表示、文字間隔、折り返し位置
- [ ] **モバイル対応**: iPhone/Android実機またはDevToolsで主要ブレークポイント確認
- [ ] **30代後半の可読性**: 小さすぎるテキストや薄すぎる色がないか

## 5. コンポーネント品質

- [ ] **shadcn/ui準拠**: カスタムスタイルではなくshadcn/uiコンポーネントを使用
- [ ] **ボタン視認性**: CTA（主要アクション）が明確に目立つ
- [ ] **ホバー/フォーカス状態**: インタラクティブ要素にhover/focusスタイルあり
- [ ] **ローディング状態**: 非同期操作にスケルトン/スピナー表示

## 6. アクセシビリティ

- [ ] **スキップリンク**: ページ先頭にメインコンテンツへのスキップリンク
- [ ] **aria属性**: アイコンボタンにaria-label、装飾画像にaria-hidden
- [ ] **キーボード操作**: Tab順序が論理的、focusリングが見える
- [ ] **alt テキスト**: 意味のある画像にalt属性

---

## 実行方法

1. Playwright MCP でページを開く
2. `browser_evaluate` でCSS変数・コントラスト比を計測
3. `browser_take_screenshot` で各セクションの視覚確認
4. 上記チェックリストを順に確認
5. 問題があれば `documents/tmp/` に報告書を作成

## カラー基準値（ライトモード）

| 変数 | 値 | 用途 |
|------|---|------|
| `--background` | `#F8FAFC` | ページ背景 |
| `--foreground` | `#0F172A` | 見出し・本文 |
| `--muted-foreground` | `#475569` | 副テキスト・説明文 |
| `--card` | `#FFFFFF` | カード背景 |
| `--border` | `#E2E8F0` | ボーダー |
| `primary-500` | `#1A56DB` | ブランド・CTA |
| `primary-700` | `#1E3A8A` | グラデーション濃い側 |
