# PFWise 実装品質 総合レビュー（5軸・100点満点）

このプロジェクト（PFWise Portfolio Manager）の実装を、以下の手順で体系的にレビューしてください。

---

## 必須: プロジェクトドキュメント読み込み（最初に実行）

レビュー開始前に以下のファイルを必ず読み込んでください：

1. **`CLAUDE.md`（プロジェクトルート）** — コーディング原則（第1条〜第10条）・アーキテクチャ概要・技術スタック・デプロイ手順
2. **`docs/review-guide.md`** — 5領域の詳細チェックリスト（テスト品質・処理正確性・マネタイズ・デザイン・ペルソナ）
3. **`docs/planning/redesign-plan.md`** — ペルソナ定義（タケシ）・収益モデル（Free/Standard）・Phase管理・ロードマップ
4. **`docs/ui-review-checklist.md`** — shadcn/ui準拠・アクセシビリティ・フィンテック信頼感デザイン詳細チェック

これらのドキュメントがレビューの判断基準となります。読み込み後にレビューを開始してください。

---

## STEP 0: レビュー対象の特定

引数が指定されている場合：その機能・ファイル・コミット範囲をレビュー対象とする。
引数がない場合：`git diff HEAD~1 HEAD --name-only` で直近のコミット変更ファイルを特定し、そのファイルをレビュー対象とする。

---

## STEP 1: PFWise プロジェクトコンテキストの読み込み

以下のファイルから必要な情報を収集してください：

### ペルソナ定義
`docs/planning/redesign-plan.md` の「ペルソナ」セクションを読む。
- プライマリ: タケシ（28〜42歳, ITエンジニア, 投資歴2〜7年, 500万〜5,000万円運用）
- ペイン: ①複数口座統合 ②リバランス計算 ③投資判断の不安

### 収益モデル
`docs/planning/redesign-plan.md` の「料金プラン」セクション：
- Free: 5銘柄, 3回/日取得, 3回/月シミュレーション, 3指標スコア, 1回/月AI
- Standard (¥700/月 or ¥7,000/年): 無制限銘柄, リアルタイム, 8指標, 無制限AI

### 技術スタック
- React 18 + TypeScript 5.x (strict: true)
- Zustand 5.x（4ストア: auth/portfolio/ui/subscription）
- TanStack Query 5.x（サーバー状態キャッシュ）
- Vitest + React Testing Library + MSW v1
- shadcn/ui + TailwindCSS（CSS変数ベース）
- Vite 7.x, Cloudflare Pages, AWS Lambda

### レビューガイド参照
`docs/review-guide.md` の詳細チェックリストを5軸レビューの補完情報として使用する。

## STEP 2: コードベース調査

Serena MCP を使って対象コードを調査：

1. `get_symbols_overview` で変更ファイルのシンボル一覧を取得
2. `find_symbol` で変更されたシンボルのbodyを読む
3. `find_referencing_symbols` で影響範囲（呼び出し元・依存先）を確認
4. テストファイルを特定し、テストカバレッジを確認

### 軸1. テスト実装品質 【配点：20点】

**PFWise固有チェックポイント:**

| 優先度 | 対象 | チェック内容 |

| P1 | サブスクリプション制限 | Free/Standard境界のテストがある（5銘柄制限等） |
| P1 | Zustandストア | `getState().action()` → 状態変化の検証テストがある |
| P2 | APIエラーハンドリング | 401→リフレッシュ、5xx→フォールバックのテストがある |

**テストパターン確認:**
- [ ] `vi.mock()` で Zustand ストアをモック（Context Provider ラップではない）
- [ ] MSW v1 ハンドラーで API をモック（`rest.get/post` 形式）

**アンチパターン:**
- [ ] `toMatchSnapshot()` 多用（振る舞いテストを優先）
- [ ] `toBeDefined()` のみ（具体値を検証していない）

| 18〜20 | P0/P1の全パス網羅、MSW/vi.mock正しく使用、カバレッジ基準超 |

### 軸2. 処理の正確性 【配点：20点】

**PFWise固有チェックポイント:**

| 計算対象 | ファイル | 検証すべき点 |
|---------|---------|-------------|
| PnL（損益） | `utils/plCalculation.ts` | 取得単価×数量 vs 現在価格×数量。通貨混在時の為替適用順序 |
| 為替変換 | `utils/currencyUtils.ts` | JPY↔USD変換の丸め方向。レート未取得時のフォールバック |
| ポートフォリオスコア | `utils/portfolioScore.ts` | 8指標の計算式がredesign-plan.md定義と一致。Free(3)/Standard(8)分岐 |
| リバランス | `utils/rebalanceCalculation.ts` | 目標配分差分計算。売買数量の切り捨て/切り上げ |

**Zustandストア整合性:**
- [ ] authStore: ログイン→JWT保存→リフレッシュ→ログアウトの状態遷移
- [ ] portfolioStore: アセット追加/削除後にサマリーが再計算される
- [ ] subscriptionStore: プラン変更後に `isPremium()` / `canUseFeature()` が即座に反映
- [ ] ログアウト時に portfolioStore のデータがクリアされる

**TanStack Query整合性:**
- [ ] キャッシュキーが一意（`['portfolio', userId]` 等）
- [ ] `staleTime` / `gcTime` が適切
- [ ] Mutation後に関連Queryが `invalidateQueries` される

**セキュリティ:**
- [ ] APIキー・JWTトークンがログ出力されていない
- [ ] `withCredentials: true` が全APIリクエストに設定
- [ ] TypeScript strict: true でコンパイルエラーなし

| 点数 | 基準 |
|------|------|
| 18〜20 | 計算正確、エラーハンドリング完備、セキュリティ問題なし、型安全 |
| 14〜17 | おおむね正確、一部エッジケースやエラー処理が不足 |
| 10〜13 | 正常系は動くが異常系で予期しない挙動が起きうる |
| 0〜9 | 計算誤り、またはセキュリティ上の問題がある |
