# PFWise ユーザーシナリオテスト監査

実際のユーザー操作シナリオ 10 項目に対し、テストカバレッジの有無と品質を監査します。ペルソナ「タケシ」が日常的に行う操作フローが、テストで保護されているかを検証します。

---

## 必須: プロジェクトドキュメント読み込み（最初に実行）

1. **`CLAUDE.md`（プロジェクトルート）** — アーキテクチャ・テスト構成・技術スタック
2. **`docs/planning/redesign-plan.md`** — ペルソナ「タケシ」定義・機能仕様
3. **`docs/review-guide.md`** — テスト品質・処理正確性の基準
4. **`MEMORY.md`（.claude/）** — 通貨変換バグ修正履歴・テスト品質アンチパターン

---

## 監査対象: 10 のユーザーシナリオ

以下の各シナリオについて、関連するソースファイルとテストファイルを特定し、カバレッジ状況を判定してください。

---

### シナリオ 1: CSV インポート（SBI/楽天/マネックス）

**ユーザー操作フロー:**
タケシが証券会社からダウンロードした CSV を取り込み、ポートフォリオに反映する。

**検証すべきソースファイル:**
- `src/` 配下の CSV パーサー関連ファイル（SBI, Rakuten, Monex, 汎用）
- `src/components/data/` 配下のインポートコンポーネント

**テストで検証すべきエッジケース:**
- [ ] SBI/楽天/マネックス各フォーマットの正常パース
- [ ] 重複ティッカーの処理（上書き or 数量加算）
- [ ] 文字コード問題（Shift_JIS / UTF-8 / BOM付き）
- [ ] 大量データ（100銘柄超）の処理
- [ ] 不正な CSV（列不足、数値でないセル、空行）
- [ ] 通貨フィールドの欠損・不正値

---

### シナリオ 2: ダッシュボード表示（JPY/USD 混在ポートフォリオ）

**ユーザー操作フロー:**
タケシがダッシュボードを開き、JPY 銘柄と USD 銘柄が混在するポートフォリオの全体像を確認する。

**検証すべきソースファイル:**
- `src/stores/portfolioStore.ts` — データ管理
- `src/utils/portfolioDataEnricher.ts` — データ加工
- `src/components/dashboard/` — 表示コンポーネント

**テストで検証すべき統合フロー:**
- [ ] store → enricher → component の一連のデータフロー
- [ ] `totalValue` が通貨変換済みの正確な合計値になっている
- [ ] 各銘柄の表示値が `baseCurrency` に統一されている
- [ ] `exchangeRate` 変更時に全値が再計算される
- [ ] 銘柄数 0 件（空ポートフォリオ）の表示

---

### シナリオ 3: 通貨トグル JPY ↔ USD

**ユーザー操作フロー:**
タケシが表示通貨を JPY から USD（またはその逆）に切り替え、全ての表示金額が即座に更新される。

**検証すべきソースファイル:**
- 通貨切替 UI コンポーネント
- `portfolioStore` の `baseCurrency` 管理
- 表示金額を算出する全コンポーネント

**テストで検証すべき内容:**
- [ ] トグル操作で `baseCurrency` が切り替わる
- [ ] ダッシュボードの総資産額が即座に再計算
- [ ] 個別銘柄の評価額が再計算
- [ ] PnL（損益）表示が再計算
- [ ] グラフの Y 軸ラベル・通貨記号が切り替わる
- [ ] 切替後にページリロードしても設定が保持される（persist）

---

### シナリオ 4: シミュレーション（クロス通貨予算）

**ユーザー操作フロー:**
タケシが「月5万円追加投資」をシミュレーションする。ポートフォリオに USD 銘柄が含まれており、予算の通貨変換が必要。

**検証すべきソースファイル:**
- `src/components/simulation/` — シミュレーション UI
- シミュレーション計算ロジック

**テストで検証すべき内容:**
- [ ] JPY 予算 → USD 銘柄への配分時に為替変換が適用
- [ ] USD 予算 → JPY 銘柄への配分時に為替変換が適用
- [ ] シミュレーション結果の合計が予算額と一致（丸め誤差考慮）
- [ ] `exchangeRate` が変動した場合の再計算
- [ ] 予算 0 円の境界ケース

---

### シナリオ 5: 目標配分 + リバランス提案

**ユーザー操作フロー:**
タケシが目標配分を設定し、現在のポートフォリオとの乖離率を確認し、リバランスの売買提案を受け取る。

**検証すべきソースファイル:**
- `src/components/settings/` — 配分設定 UI
- リバランス計算ロジック
- `src/utils/` 配下の乖離率計算

**テストで検証すべき内容:**
- [ ] 目標配分の合計が 100% になるバリデーション
- [ ] 乖離率（drift）の計算が通貨変換済みの値で行われている
- [ ] 売買提案の金額が具体的かつ正確
- [ ] JPY/USD 混在ポートフォリオでのリバランス計算
- [ ] 目標配分未設定時の表示

---

### シナリオ 6: エクスポート（CSV/JSON/PDF）

**ユーザー操作フロー:**
タケシがポートフォリオデータを CSV/JSON/PDF 形式でエクスポートする。

**検証すべきソースファイル:**
- `src/components/data/` — エクスポート UI
- エクスポートロジック（CSV生成、JSON生成、PDF生成）

**テストで検証すべき内容:**
- [ ] CSV エクスポートのヘッダー行・データ行が正確
- [ ] JSON エクスポートのスキーマが正確
- [ ] PDF エクスポートの内容が正確（「クラッシュしない」だけでは不十分）
- [ ] エクスポートデータに通貨情報が含まれている
- [ ] 銘柄数 0 件でのエクスポート挙動
- [ ] Free プランの形式制限（CSV のみ）が適用されている

**テストで検証すべきフルフロー:**

- [ ] 価格変動 → 閾値判定 → 通知トリガー
- [ ] 通知の表示と既読処理
- [ ] 通貨変換が必要なアラートの閾値判定
- [ ] アラートの編集・削除

## 判定基準

各シナリオを以下の 3 段階で判定してください：

| 判定 | 基準 |
|------|------|
| **COVERED** | メインフロー＋主要エッジケースがテストされている。テストファイルを特定できる |
| **PARTIAL** | メインフローのテストはあるが、重要なエッジケースが不足。具体的に何が足りないか記述 |
| **MISSING** | テストが存在しない、またはレンダリングチェックのみで実質的なカバレッジなし |

## 注意事項

- テストファイルの内容を必ず実際に読み、アサーションの品質まで確認すること（ファイル名だけで COVERED と判定しない）
- `toBeGreaterThan(0)` や `toBeDefined()` だけのテストは COVERED と判定しない
- 通貨変換に関わるシナリオ（2, 3, 4, 5）は MEMORY.md のバグ修正履歴と照合すること
- Serena MCP を活用してシンボル構造・依存関係を効率的に分析すること

ARGUMENTS: （特定のシナリオ番号を指定可能。例: "1,3,5" で CSV インポート・通貨トグル・リバランスのみ監査。省略時は全 10 シナリオを監査）
